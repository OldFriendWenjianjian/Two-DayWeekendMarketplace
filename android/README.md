# 双休超市 Android Shell

Android WebView shell for the 双休超市 blockchain marketplace.

## What It Loads

By default, the app opens:

```text
http://[2402:4e00:c013:8600:5602:3dc2:a2d0:0]/shc-20260520-a1faaf/weekend-marketplace/
```

If the main-frame remote page fails to load, it falls back to the bundled asset:

```text
file:///android_asset/www/index.html
```

The launcher activity also accepts a supported `http`, `https`, or `file` URL from an intent data URI, or from the `start_url` string extra.

## Capabilities

- JavaScript and DOM storage enabled.
- Mixed content and cleartext HTTP allowed for the current IPv6 HTTP endpoint.
- Media playback does not require a user gesture.
- Runtime permissions requested for camera and microphone.
- WebRTC `getUserMedia` permission requests are granted after Android runtime permission checks.
- File chooser supports Android document pickers and camera capture via `FileProvider`.
- Generated launcher icons are included in all Android density buckets and adaptive icon XML points at the generated foreground asset.

## Build

This directory is a standard Gradle Android project. On this machine the verified command is:

```powershell
cd C:\Users\a1258\Documents\Two-DayWeekendMarketplace\android
$env:JAVA_HOME='C:\Program Files\Android\Android Studio\jbr'
$env:ANDROID_HOME='C:\Users\a1258\AppData\Local\Android\Sdk'
$env:ANDROID_SDK_ROOT=$env:ANDROID_HOME
& 'C:\Users\a1258\.gradle\wrapper\dists\gradle-8.7-bin\bhs2wmbdwecv87pi65oeuq5iu\gradle-8.7\bin\gradle.bat' --no-daemon :app:assembleDebug
```

If Android Studio is used, open the `android/` directory as the project root and let it sync the Gradle project.

The project currently expects:

- Android Gradle Plugin `8.5.2`
- `compileSdk 34`
- Java 17
- Package name `com.twodayweekend.marketplace`
