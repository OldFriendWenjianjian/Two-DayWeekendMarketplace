# 双锚信誉共识手册

## 一句话

双休超市不是“谁账号多，谁就有理”。账号可以自由注册，但治理权不能凭空产生。

## 为什么需要它

双休超市允许每个人成为买家，也允许顾客申请成为店家。店家 ID 是个人品牌，登记后全局唯一、不可重复创建、不可删除。评价、投诉、治理下架等信誉事件会进入私有签名账本，形成可同步、可校验、不可篡改的履历。

如果一个系统只看账号数量、账号年龄或简单投票，攻击者可以批量注册新号，甚至提前养号几年，再集中攻击一个商户。双锚信誉共识的目标，就是让空号再多也不能伤害任何人的核心信誉。

## 核心原则

一万个空号 = 0 治理权。

普通留言可以被看见，但不直接毁掉任何人的店铺信誉。

真正影响核心信誉的行为，必须至少具备以下一个锚点：

- 真实交易锚：评价者和商品之间存在已完成订单。
- 责任押注锚：评价者愿意押上自己的信誉、保证金或获得足够见证人背书。

评价别人，也是在书写自己的履历。每次投诉、评价、下架申请都会同时记录在发起人和被评价对象的账本历史中。如果事后被判定为恶意攻击，发起人自己的后续治理权会被削弱，严重时清零。

## 四类行为

普通留言：可见、可追溯、会留在双方历史中，但治理权重为 0，不影响核心信誉。

订单评价：只有绑定已完成订单时，才形成真实交易锚，可以影响商户核心信誉。

责任投诉：没有订单也可以投诉，但必须有责任押注或足够见证人，才会进入核心治理。否则只作为普通反馈留痕。

下架申请：只有真实交易锚或责任押注锚成立时，才进入治理队列；没有锚点时只是 0 权重反馈。

## 防攻击逻辑

新号没有完成订单，没有押注，也没有见证背书时，权重为 0。

养号不自动增加权重。账号年龄只可作为展示信息，不作为治理权来源。

批量账号不能叠加权重。每个空号都是 0，十万个空号相加仍然是 0。

恶意投诉会反噬发起人。系统会把被驳回、被证明造假的治理行为写入发起人履历，并降低后续共识权。

见证人也要负责。见证背书不是免费按钮，见证人同样会在自己的履历里留下记录。

## 面向商户的承诺

双休超市欢迎用户监督，但不鼓励情绪化围攻。商户不会因为一批陌生空号的差评而被直接打垮。真正有影响的评价，需要来自真实交易、责任押注或社区见证。

## 面向用户的承诺

用户可以自由表达，也可以推动治理。但越严肃的指控，越需要承担相应责任。系统保护认真反馈的人，也约束恶意攻击的人。

## 宣传文案

不是谁账号多，谁就有理。

账号自由注册，信誉不能凭空制造。

普通留言可以被看见，但不会毁掉任何人。

真实交易锚 + 责任押注锚，让评价有证据，也有担当。

评价别人，也是在书写自己的信誉。

双休超市支持好产品，也支持做出好产品的人按时下班。
